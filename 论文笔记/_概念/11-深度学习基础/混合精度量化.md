# 混合精度量化（Mixed-Precision Quantization）

## 定义

混合精度量化（MPQ）是网络量化的一种策略，对模型中不同层、不同通道或不同 token 分配不同的比特宽度（bit-width），以在总体量化误差和计算/存储开销之间取得更优的权衡，而非对全网络统一使用同一精度。

## 数学形式

在约束总比特运算（BOPs）预算 $\mathcal{B}$ 的条件下，最小化各层量化误差之和：

$$
\mathbf{b}^* = \arg\min_{\mathbf{b}} \sum_{l=1}^{L} \mathcal{L}_l(b_l) \quad \text{s.t.} \quad \text{BOP}(\mathbf{b}) \leq \mathcal{B}
$$

其中 $\mathbf{b} = \{b_l\}_{l=1}^L$ 为各层比特宽度向量，$\mathcal{L}_l(b_l)$ 为第 $l$ 层在 $b_l$ bits 下的量化误差。

## 核心要点

1. **粒度分类**:
   - **层级（layer-wise）**: 不同层用不同 bits（最常见）
   - **通道级（channel-wise）**: 同层内不同 channel 用不同 bits
   - **Token 级（token-wise）**: 同层内不同 token 用不同 bits（如 [[TokenBitViT]]）
2. **比特宽度搜索**: 通常用强化学习、NAS、整数线性规划（ILP）或启发式方法确定各层比特宽度
3. **效率指标**: 常用 BOPs（比特运算次数）或模型大小作为约束，而非单纯最小化误差
4. **敏感性分析**: 对量化误差敏感的层分配高精度（如分类头附近层），不敏感的层分配低精度（如早期卷积层）
5. **动态 vs 静态**: 静态 MPQ 在推理前固定比特宽度；动态 MPQ（如 [[TokenBitViT]]）在推理时根据输入实例动态决定

## 与 ViT 量化的关系

[[Vision Transformer|ViT]] 的激活分布在不同 token 间差异显著，是引入 token 级动态 MPQ 的核心动机：
- 固定比特宽度方案对难量化 token（动态范围大）精度损失大
- [[TokenBitViT]] 用量化区间近似误差，以极低开销实现 token 级动态比特分配

## 代表工作

- HAQ (CVPR 2019): 用强化学习搜索混合精度方案
- ENAS-Q (NeurIPS 2020): NAS 驱动的混合精度搜索
- [[IGQ-ViT]] (CVPR 2024): ViT channel 分组量化，自适应分配 group 数（间接 MPQ）
- [[TokenBitViT]] (PR 2025): ViT token 级动态比特宽度分配

## 相关概念

- [[训练后量化]]: MPQ 常在 PTQ 框架下实现
- [[Vision Transformer]]: ViT 的 token 间分布差异是 MPQ 的重要应用场景
- [[Activation Quantization]]: 激活值的 MPQ 是核心挑战
- [[Group Quantization]]: 分组量化是 channel 级 MPQ 的一种实现
